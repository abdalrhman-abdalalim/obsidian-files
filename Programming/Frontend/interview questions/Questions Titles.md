
### إيه الفرق بين em, rem, و px؟ وامتى تستخدم كل واحد؟
- **px** → لما تكون محتاج دقة ثابتة (زي borders، خطوط دقيقة، أو عناصر جرافيكية).
    
- **em** → لما تحب العنصر يتأثر بالـ parent (زي padding أو margin مبني على حجم خط العنصر نفسه).
    
- **rem** → لما تحب consistency وسهولة scaling (أفضل للـ typography، spacing الأساسي في الـ layout).

### إزاي الـ Critical Rendering Path بيأثر على سرعة تحميل الصفحة؟

هو **الخطوات اللي المتصفح بيعملها عشان يحول الكود (HTML, CSS, JS)** لصفحة مرئية يقدر الـ User يشوفها ويتفاعل معاها.
#### مراحل الـ CRP:

1. **Parse HTML → DOM**  
    المتصفح يقرأ الـ HTML ويبني **DOM tree**.
    
2. **Parse CSS → CSSOM**  
    يقرأ ملفات الـ CSS ويبني **CSSOM tree**.
    
3. **Combine → Render Tree**  
    يدمج الـ DOM + CSSOM → يطلع **Render Tree** (العناصر اللي هتظهر + الـ styles بتاعتها).
    
4. **Layout (Reflow)**  
    يحدد أماكن وأحجام العناصر على الصفحة.
    
5. **Paint → Composite**  
    يرسم العناصر على الشاشة.



### إزاي تتجنب الـ Cumulative Layout Shift (CLS) وتحسن تجربة المستخدم؟

هو مقياس بيحسب **قد إيه عناصر الصفحة بتتنطط أو بتتحرك فجأة أثناء التحميل**.  
مثال:

- تكون بتقرأ نص → فجأة صورة تتحمّل وتزق النص لتحت.
- أو زرار يظهر فجأة ويتحرك مكان زرار تاني → المستخدم يضغط حاجة غلط.

####  إزاي تتجنب CLS؟
1. **ثبّت مساحة الصور والفيديوهات**
2. **احجز مكان للإعلانات والـ embeds**. 
3. **حمّل الخطوط بشكل ذكي**
4. **تجنّب إضافة عناصر فوق المحتوى**
5. **استخدم animations/transitions صح**
